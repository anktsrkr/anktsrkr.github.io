Now, we know the context of this topology and architecture let’s start to implement it. In this post, I am going to use azure powershell module to create all azure resources.

In my second post of this series we have already created hub-vnet.

Let’s create Spoke 1 Virtual Network(spoke1-vnet), with address space 10.20.0.0/19. We will also create a subnet WorkloadSubnet under Spoke1 virtual network with address space 10.20.1.0/24

 1$LocationName = "centralindia"
 2$ResourceGroupName = "network-sandbox"
 3$vnetName = "spoke1-vnet"
 4$defaultSubnet = "WorkloadSubnet"
 5$vnetAddressSpace = "10.20.0.0/19"
 6$subnetvnetAddressSpace = "10.20.1.0/24"
 7
 8$virtualNetwork = New-AzVirtualNetwork `
 9                     -ResourceGroupName $ResourceGroupName `
10                     -Location $LocationName `
11                     -Name $vnetName `
12                     -AddressPrefix $vnetAddressSpace
13
14$subnetConfig = Add-AzVirtualNetworkSubnetConfig `
15                   -Name $defaultSubnet `
16                   -AddressPrefix $subnetvnetAddressSpace `
17                   -VirtualNetwork $virtualNetwork
18
19
20$virtualNetwork = $virtualNetwork | Set-AzVirtualNetwork

Please note, by default DDoS protection plan as Basic is selected, but it is highly recommended to enable DDoS protection plan for your production environment and this is applicable for all of your virtual networks.

Next create Spoke 2 Virtual Network(spoke2-vnet), with address space 10.30.0.0/19. We will also create a subnet WorkloadSubnet under Spoke1 virtual network with address space 10.30.1.0/24

Two virtual networks can not talk to each other, so by default hub-vnet will not be able to connect with spoke1-vnet and spoke2-vnet. So, to make it work we need to enable vnet-peering between hub and spokes.

Let’s create vnet peering between hub and spokes:

 1$hubvnetName = "hub-vnet"
 2$hubVnet = Get-AzVirtualNetwork `
 3              -Name $hubvnetName
 4
 5$spoke1vnetName = "spoke1-vnet"
 6$spoke1Vnet = Get-AzVirtualNetwork `
 7                 -Name $spoke1vnetName
 8
 9$spoke2vnetName = "spoke2-vnet"
10$spoke2Vnet = Get-AzVirtualNetwork `
11                 -Name $spoke2vnetName
12
13
14# Peer Hub to Spoke 1.
15Add-AzVirtualNetworkPeering -Name 'LinkHubToSpoke1' `
16                            -VirtualNetwork $hubVnet `
17                            -RemoteVirtualNetworkId $spoke1Vnet.Id
18
19# Peer Spoke 1 to Hub.
20Add-AzVirtualNetworkPeering -Name 'LinkSpoke1ToHub' `
21                            -VirtualNetwork $spoke1Vnet `
22                            -RemoteVirtualNetworkId $hubVnet.Id
23
24# Peer Hub to Spoke 2.
25Add-AzVirtualNetworkPeering -Name 'LinkHubToSpoke2' `
26                            -VirtualNetwork $hubVnet `
27                            -RemoteVirtualNetworkId $spoke2Vnet.Id
28
29# Peer Spoke 2 to Hub.
30Add-AzVirtualNetworkPeering -Name 'LinkSpoke2ToHub' `
31                            -VirtualNetwork $spoke2Vnet `
32                            -RemoteVirtualNetworkId $hubVnet.Id

NOTE: You can also provide resource id if you do not have read access to the virtual network or subscription you wish to peer with.

Once done, peering status should show as connected :

The interesting part is I have created peering between hub and spokes but not between spoke1 and spoke2, so spoke1 and spoke2 can not communicate with each other. you can ask me, can’t we create peering between them? Yes! you can but you should not, I will try to explain the reason in next part and will show you how communication will be done between spokes if needed.

Once completed, Connect you VPN and let’s see how data flows between hub to spokes from you local workspace.

Local workspace To Hub

Since, we have established the VPN connection, Hub is reachable from local workspace.

Hub To Spokes

Since, we have established the peering between Hub and spokes, these spokes are reachable from hub.

Spokes To Hub

Hub is also reachable from both spokes.

Local workspace To Spokes

Spokes are not reachable from local workspaces. It supposed to work right? To make it work, you need to perform some more steps -

  1. Configure all peering connections to allow forwarded traffic.

  2. Configure the peering connection in the hub to allow gateway transit. 

     1$resourceGroupName = "network-sandbox"
 2$hubvnetName = "hub-vnet"
 3
 4# Update Hub to Spoke 1.
 5$LinkHubToSpoke1 = Get-AzVirtualNetworkPeering `
 6                      -VirtualNetworkName $hubvnetName `
 7                      -ResourceGroupName $resourceGroupName `
 8                      -Name "LinkHubToSpoke1"
 9
10$LinkHubToSpoke1.AllowGatewayTransit = $True
11$LinkHubToSpoke1.AllowForwardedTraffic = $True
12
13Set-AzVirtualNetworkPeering -VirtualNetworkPeering $LinkHubToSpoke1
14
15
16# Update Hub to Spoke 2.
17$LinkHubToSpoke2 = Get-AzVirtualNetworkPeering `
18                      -VirtualNetworkName $hubvnetName `
19                      -ResourceGroupName $resourceGroupName `
20                      -Name "LinkHubToSpoke2"
21
22$LinkHubToSpoke2.AllowGatewayTransit = $True
23$LinkHubToSpoke2.AllowForwardedTraffic = $True
24
25Set-AzVirtualNetworkPeering -VirtualNetworkPeering $LinkHubToSpoke2

  3. Configure the peering connection in each spoke to use remote gateways. 

     1$resourceGroupName = "network-sandbox"
 2# Update Spoke 1 to Hub.
 3
 4$spoke1vnetName = "spoke1-vnet"
 5$LinkSpoke1ToHub = Get-AzVirtualNetworkPeering `
 6                      -VirtualNetworkName $spoke1vnetName `
 7                      -ResourceGroupName $resourceGroupName `
 8                      -Name "LinkSpoke1ToHub"
 9
10$LinkSpoke1ToHub.UseRemoteGateways = $True
11$LinkSpoke1ToHub.AllowForwardedTraffic = $True
12
13Set-AzVirtualNetworkPeering -VirtualNetworkPeering $LinkSpoke1ToHub
14
15# Update Spoke 2 to Hub.
16
17$spoke2vnetName = "spoke2-vnet"
18$LinkSpoke2ToHub = Get-AzVirtualNetworkPeering `
19                      -VirtualNetworkName $spoke2vnetName `
20                      -ResourceGroupName $resourceGroupName `
21                      -Name "LinkSpoke2ToHub"
22
23$LinkSpoke2ToHub.UseRemoteGateways = $True
24$LinkSpoke2ToHub.AllowForwardedTraffic = $True
25
26Set-AzVirtualNetworkPeering -VirtualNetworkPeering $LinkSpoke2ToHub
